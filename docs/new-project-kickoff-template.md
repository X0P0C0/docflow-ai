# 新项目启动模板

## 使用方式
这个模板用于在项目正式编码前，先完成最基础的方向确认、范围收敛和工程准备。

建议规则：
- 先填这份文档，再开始写代码
- 没定清楚的地方先标记 `待确认`
- 第一版不追求面面俱到，重点是把边界和顺序定住

## 推荐开场指令
如果你以后想让我陪你从零开始做一个项目，可以直接对我说：

```text
我们现在要启动一个新项目，请先不要写代码，先帮我完成这些事情：
1. 明确项目目标和非目标
2. 梳理角色与核心业务流程
3. 拆出 MVP 范围
4. 设计领域模型和接口边界
5. 给出技术选型和工程规范
6. 生成项目总清单文档
等这些完成后，再开始进入编码阶段
```

---

## 最后提醒
正确的项目启动方式不是：

`想到什么做什么`

而是：

`先定目标 -> 再定边界 -> 再定顺序 -> 最后进入实现`
