# DocFlow AI 工程化建设清单

## 说明
这份文档用于沉淀 `DocFlow AI` 项目的工程化建设范围、当前现状、优先级和后续动作。

目标不是一次性“全部做完”，而是建立一个稳定的改造顺序，避免需求、代码、页面、联调、部署各自为战。

建议使用方式：
- 先按 `P0 -> P1 -> P2` 推进，不要并行开太多口子
- 每完成一项，就更新本文档的状态
- 新增功能前，先确认它影响的是哪一个模块

状态说明：
- `未开始`：还没有落地
- `进行中`：已经开始治理，但还没闭环
- `已完成`：已有明确实现和验证

---

## 3. 接口规范
### 目标
保证前后端交互格式、错误响应、分页、筛选、排序规则一致。

### 应包含内容
- 统一响应结构
- 统一错误码
- 统一参数校验返回
- 分页格式
- 查询筛选规范
- 幂等与重复提交策略

### 当前现状
- 已有统一 `ApiResponse`
- 已有基础全局异常处理
- 已补充统一错误标识 `error`、请求路径 `path`、时间戳 `timestamp`、校验明细 `errors`、链路标识 `traceId`
- 错误码字典和业务细分错误码仍需继续扩充

### 当前状态
`进行中`

### 优先级
`P0`

### 后续动作
- 继续扩充错误码字典
- 统一 bad request / business error / system error 响应格式
- 收敛接口命名、分页和筛选参数规范

`进行中`

`P0`

### 应包含内容

- Token 生命周期
- 默认账号与默认密钥治理

- 接口暴露面控制
- Swagger 暴露策略

### 当前现状
- 已有 JWT、Spring Security、角色权限判断
- 默认密钥和环境隔离策略仍需收口
- 接口脱敏、审计、安全加固还不足

### 当前状态
`进行中`

### 优先级
`P0`

### 后续动作
- 清理默认密钥兜底策略
- 收口 Swagger 暴露条件
